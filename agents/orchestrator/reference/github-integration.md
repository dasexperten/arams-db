# github-integration.md

Reference for the Level 3 auto-merge pipeline — how Apps Script talks to
GitHub, branch/commit/PR/merge lifecycle, validation gates, and failure modes.

---

## Authentication

All GitHub API requests use a **fine-grained Personal Access Token (PAT)** stored
exclusively in Script Property `GITHUB_PAT`. The token is fetched at request time —
never cached in memory across executions, never logged.

Required token permissions (repository-scoped, single repo):
- **Contents:** Read and Write — create branches, commit files
- **Pull requests:** Read and Write — open and merge PRs
- **Metadata:** Read — resolve repo state

```javascript
// Every API call goes through githubAuth_() — token stays in header, never in logs
function githubAuth_() {
  var token = getProp_('GITHUB_PAT', true);
  var repo  = getProp_('GITHUB_REPO', true); // format: owner/repo-name
  return {
    headers: { Authorization: 'Bearer ' + token, Accept: 'application/vnd.github+json',
               'X-GitHub-Api-Version': '2022-11-28' },
    repo: repo,
    apiBase: 'https://api.github.com/repos/' + repo
  };
}
```

`redactToken_()` strips the token value from any string before it reaches
`console.log` / `console.warn`. Use it on all error messages that may
include raw API response bodies.

## API endpoints used

| Operation | Method | Endpoint |
|---|---|---|
| Get HEAD SHA of branch | GET | `/repos/{owner}/{repo}/git/ref/heads/{branch}` |
| Create feature branch | POST | `/repos/{owner}/{repo}/git/refs` |
| Check file existence | GET | `/repos/{owner}/{repo}/contents/{path}?ref={branch}` |
| Commit file | PUT | `/repos/{owner}/{repo}/contents/{path}` |
| Create PR | POST | `/repos/{owner}/{repo}/pulls` |
| Get PR state | GET | `/repos/{owner}/{repo}/pulls/{pr_number}` |
| Merge PR | PUT | `/repos/{owner}/{repo}/pulls/{pr_number}/merge` |
| Delete branch | DELETE | `/repos/{owner}/{repo}/git/refs/heads/{branch}` |

All calls use `muteHttpExceptions: true` so HTTP errors surface as parseable
JSON bodies rather than Apps Script exceptions. Status codes are checked explicitly.

## Branch naming convention

```
auto-templates/<wf_id>-<short-slug>
```

Where:
- `wf_id` — full workflow instance ID (e.g. `AUTODETECT_20260427_01`)
- `short-slug` — pattern name in lowercase, hyphens, max 30 chars

Examples:
- `auto-templates/AUTODETECT_20260427_01-distributor-followup`
- `auto-templates/AUTODETECT_20260501_02-inbox-triage-emea`

Branch names never contain spaces, underscores (except in wf_id prefix), or
special characters other than hyphens and forward slashes.

  Chat ID: [redacted]

## Pre-merge validation gates

Run by `githubValidateBeforeMerge_()` before calling merge endpoint.
Returns `{ valid: true/false, errors: [] }`.

| Check | What it verifies | Failure example |
|---|---|---|
| **File path guard** | Path starts with `agents/orchestrator/workflows/` and ends with `.md` | Malformed path injected via edited draft |
| **File size** | Content length ≤ 50 KB | Unexpectedly large generated template |
| **YAML frontmatter** | File begins with `---`, contains valid `name:` and `trigger_phrases:` keys | Missing frontmatter entirely |
| **No null bytes** | Content passes `indexOf('\0') === -1` | Corrupted base64 decode |
| **No hardcoded secrets** | No `GITHUB_PAT`, `TELEGRAM_BOT_TOKEN`, `ANTHROPIC_API_KEY` substrings in content | Accidental inclusion of env vars in generated draft |
| **Mergeable state** | PR `mergeable` field is `true` (polled up to 5s after creation — GitHub sets this async) | Merge conflict with existing file on main |

If ALL pass → `{ valid: true, errors: [] }` → proceed to merge.
If ANY fail → `{ valid: false, errors: ['File path guard failed: ...', ...] }` →
PR stays open, Aram notified via Telegram Type 5 (Urgent) with PR link.

## Conflict handling

Before committing to the feature branch, `githubGetFileContent_()` checks if
the target path already exists on `main`:

```
GET /repos/{owner}/{repo}/contents/agents/orchestrator/workflows/<name>.md
```

- **404** → file does not exist → safe to proceed
- **200** → file already exists at this path

On conflict (200):
- Do NOT proceed with commit
- Hard HALT to Aram:

```
🤖 ORCHESTRATOR — GitHub/Conflict 🔴
Шаблон "<name>" уже существует в репозитории.
Путь: agents/orchestrator/workflows/<name>.md

Варианты:
[Переименовать и попробовать снова]  [Отменить]
wf_id: <wf_id>
```

On "Переименовать": Aram provides new name via free text → orchestrator retries
with `<name>-v2.md` (or the name Aram provides).

## Network retry logic

```javascript
function retryWithBackoff_(fn, maxAttempts) {
  maxAttempts = maxAttempts || 3;
  var delays = [1000, 2000, 4000]; // ms
  var lastErr;
  for (var i = 0; i < maxAttempts; i++) {
    try {
      return fn();
    } catch (e) {
      lastErr = e;
      if (i < maxAttempts - 1) Utilities.sleep(delays[i]);
    }
  }
  throw lastErr;
}
```

Applied to: branch creation, file commit, PR creation, merge, branch deletion.
NOT applied to: pre-flight checks (conflict detection, validation) — these are
fast reads where retrying a 404 won't change anything.

After 3 failed attempts → throw → caller catches → `handleFatal_` or
`reportWorkflowError_` → HALT to Aram with full error text.

## Token rotation

`tokenRotationReminder()` fires daily at 09:00 (via Apps Script time trigger).

Logic:
1. Read `GITHUB_PAT_ISSUED_DATE` from Script Properties (ISO date string, set
   manually when the operator creates a new token).
2. Compute `days_since_issue = (now - issued_date) / 86400000`.
3. If `days_since_issue >= 80`:
   - Send Telegram reminder (Type 5 — Urgent) with link to GitHub token settings.
   - Do NOT disable GitHub operations — token may still be valid.
4. If `days_since_issue >= 90`:
   - Send second Telegram: "Token is likely expired. GitHub operations will fail."
   - Log `error_type: token_expired_suspected` in Script execution log.

After operator rotates the token:
1. Update `GITHUB_PAT` Script Property.
2. Update `GITHUB_PAT_ISSUED_DATE` to today's date (ISO string).

Token itself is never stored in Drive, Sheet, or any log — only in Script Properties.

## Failure modes table

| Failure | HTTP status | Orchestrator action |
|---|---|---|
| Token expired or revoked | 401 | HALT immediately. Telegram: "GITHUB_PAT expired. Update Script Property." No retry. |
| Insufficient token scope | 403 | HALT. Telegram: "Token permissions insufficient for <operation>. Check repo access." |
| Rate limit hit | 403 + `X-RateLimit-Remaining: 0` | Retry after `X-RateLimit-Reset` timestamp (if ≤ 120s). Else HALT. |
| Branch already exists | 422 | Append `-r<N>` suffix to branch name and retry once. |
| Commit conflict (PR file changed by someone else) | 409 | HALT. Telegram: "Commit conflict on <path>. Manual resolve required." |
| PR creation fails | 4xx/5xx | Retry with backoff × 3. If all fail: HALT with error body. |
| Merge blocked (no auto-merge enabled on repo) | 405 | Telegram: "Auto-merge not enabled. Enable in GitHub repo settings or merge manually." Link to PR. |
| Merge conflict | 409 | PR stays open. Telegram: "Merge conflict detected. Resolve in GitHub and merge manually." |
| Validation failed | N/A (local check) | PR stays open. Telegram with error list + PR link. Not a HALT — PR is accessible for manual review. |
| Branch delete fails post-merge | 4xx | Non-fatal. Log warning. Branch cleanup can be done manually. |
| Apps Script 6-minute timeout mid-pipeline | N/A | State checkpointed before each API call. Resume picks up from last checkpoint. |

## Full pipeline sequence (`githubFullPipeline_`)

```
1. Check conflict (GET contents/<path> on main)
   → 200: HALT (conflict)
   → 404: continue

2. Get HEAD SHA of main
   → retryWithBackoff_

3. Create feature branch auto-templates/<wf_id>-<slug>
   → retryWithBackoff_

4. Base64-encode template content
   → Utilities.base64Encode(content)

5. Commit file to feature branch
   → retryWithBackoff_

6. Create PR (title, body with full audit metadata)
   → retryWithBackoff_
   → Store pr_number and pr_url in state.data

7. Poll PR mergeable state (up to 5 seconds, 1s intervals)

8. Validate (githubValidateBeforeMerge_)
   → errors: keep PR open, notify Aram
   → valid: continue

9. Merge PR (squash)
   → retryWithBackoff_
   → Store merge_sha in state.data

10. Delete feature branch
    → best-effort (non-fatal on failure)

11. Delete Drive draft from pending-templates/
    → DriveApp file delete

12. Update Sheet index row: github_pr_url, github_merge_sha, validation_status
```

Total expected time: 5–15 seconds from Aram's tap to merged commit on main.
