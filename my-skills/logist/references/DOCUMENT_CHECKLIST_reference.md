## 🧭 ЛОГИКА ВЫЗОВА ЧЕКЛИСТА (для logist)

Когда logist начинает новый шипмент:

**Step 0 — Определение схемы.** Применить правило маршрутизации (SELLER ROUTING в SKILL.md):
- Товар (паста / щётка / смешанный)?
- Рынок (RU/CIS / международка)?
- → Выбрать Scheme A / B / C.

**Step 1 — Загрузка чеклиста.** Взять соответствующую таблицу из этого файла. Для смешанных грузов (пасты + щётки одним шипментом) — работать с двумя чеклистами параллельно (A + B).

**Step 2 — Создание Shipment Timeline.** Это **mandatory** для любого шипмента (см. SHIPMENT_TIMELINE_TEMPLATE.md и соответствующее правило в SKILL.md) — каждый документ из чеклиста маппится на этап маршрута.

**Step 3 — Gate integration.** Параллельно с чеклистом:
- `[[GATE: legalizer?section=certifications&sku=<SKU>]]` — проверка статуса ДС на все SKU в шипменте
- `[[GATE: contacts?entity=<slug>]]` — подтягивание реквизитов продавца и покупателя
- `[[GATE: pricer?list=PL-PRCH_CNY]]` (или USD) — закупочные цены для IS

**Step 4 — Трекер прогресса.** По каждому документу — статус (не начат / в работе / получен / утверждён / передан брокеру).

## 📎 СПЕЦИФИЧЕСКИЕ ПОДСКАЗКИ ПО ДОКУМЕНТАМ

### Invoice-Specification (IS) — ключевой документ

IS — это **внутренний российский документ** для Дениса Панченко (брокер) и ФТС РФ. Сочетает инвойс + упаковочный лист + спецификацию в одном bilingual документе.

- **Variant 1** (щётки): три стороны — Shipper (Jinxia) + Seller (DEI) + Buyer/Consignee (DEE). Цены в USD (FOB).
- **Variant 2** (пасты): две стороны — Shipper=Seller (Honghui) + Consignee=Buyer (DEE). Цены в CNY.

Генерируется через invoicer skill (Step 4, секция C).

### CNY-счёт DEE для платежей в Китай

При платежах DEE → Honghui (контракт № 080824) или DEE → Jinxia (контракт № MF01-DEA/YZ) **обязательно** использовать юаневый счёт DEE:

- **Текущий счёт (CNY/USD):** 40702156600340000037
- **Банк:** ВТБ (ПАО), БИК 044525411, SWIFT VTBRRUM2MS2
- **Route:** A (санкционный, только RU/CIS → Китай)

**Не использовать** рублёвый счёт 40702810024370000534 для международных платежей.

### Заявление на перевод ВТБ

Генерируется в logist skill перед оплатой. Обязательные поля:
- УНК контракта (из таблицы в SKILL.md, раздел ENTITY BANK DETAILS / DEE)
- Реквизиты получателя (тянуть через `[[GATE: contacts?entity=guangzhou-honghui&payer=dee]]` или `=yangzhou-jinxia&payer=dee`)
- Ссылка на контракт и инвойс

### Инструкция брокеру

Стандартный пакет для Дениса Панченко включает:
- CI (фабрика → покупатель или DEI → DEE)
- PL (фабрика)
- IS (Variant 1 или 2)
- CO (Certificate of Origin)
- ДС на все SKU
- MSDS (если пасты)
- Экспортная декларация Китая
- Страховой полис
- CMR/BoL

Передаётся за 3-5 дней до прибытия на границу, чтобы брокер успел подать предварительную декларацию.
